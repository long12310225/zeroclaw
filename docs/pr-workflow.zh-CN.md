# ZeroClaw PR工作流程（高容量协作）

本文档定义了ZeroClaw如何在保持以下方面的同时处理高PR量：

- 高性能
- 高效率
- 高稳定性
- 高可扩展性
- 高可持续性
- 高安全性

相关参考：

- [`docs/README.zh-CN.md`](./README.zh-CN.md) 了解文档分类法和导航。
- [`docs/ci-map.zh-CN.md`](./ci-map.zh-CN.md) 了解每个工作流程的所有权、触发器和分类流程。
- [`docs/reviewer-playbook.zh-CN.md`](./reviewer-playbook.zh-CN.md) 了解日常审阅者执行。

## 0. 概要

- **目的：** 为高吞吐量协作提供确定性的、基于风险的PR操作模型。
- **受众：** 贡献者、维护者和代理辅助审阅者。
- **范围：** 仓库设置、PR生命周期、准备就绪契约、风险路由、队列纪律和恢复协议。
- **非目标：** 替代分支保护配置或CI工作流程源文件作为实施权威。

---

## 1. 按PR情况快速路径

使用本节在全面深入审阅前快速路由。

### 1.1 接收入口不完整

1. 在一个检查清单评论中请求模板完成和缺失证据。
2. 在解决接收入口障碍之前停止深入审阅。

### 1.2 `CI必需门`失败

1. 通过CI地图路由失败并首先修复确定性门。
2. 仅在CI返回连贯信号后重新评估风险。
