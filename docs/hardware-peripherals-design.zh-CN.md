# 硬件外设设计 —— ZeroClaw

ZeroClaw使微控制器（MCU）和单板计算机（SBC）能够**动态解释自然语言命令**，生成硬件特定代码，并实时执行外设交互。

## 1. 愿景

**目标：** ZeroClaw作为一个硬件感知的AI代理：
- 通过通道（WhatsApp、Telegram）接收自然语言触发器（例如"移动X臂"、"打开LED"）
- 获取准确的硬件文档（数据手册、寄存器映射）
- 使用LLM（Gemini、本地开源模型）合成Rust代码/逻辑
- 执行逻辑以操作外设（GPIO、I2C、SPI）
- 保存优化代码以供未来重用

**心理模型：** ZeroClaw = 理解硬件的大脑。外设 = 它控制的手臂和腿。

## 2. 两种操作模式

### 模式1：边缘原生（独立）

**目标：** 支持WiFi的开发板（ESP32、树莓派）。

ZeroClaw**直接在设备上**运行。开发板启动gRPC/nanoRPC服务器并与外设本地通信。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  ESP32 / 树莓派上的ZeroClaw（边缘原生）                                     │
│                                                                             │
│  ┌─────────────┐    ┌──────────────┐    ┌─────────────────────────────────┐ │
│  │ 通道        │───►│ 代理循环     │───►│ RAG：数据手册、寄存器映射       │ │
│  │ WhatsApp    │    │ （LLM调用）  │    │ → LLM上下文                     │ │
│  │ Telegram    │    └──────┬───────┘    └─────────────────────────────────┘ │
│  └─────────────┘           │                                                 │
│                            ▼                                                 │
│  ┌─────────────────────────────────────────────────────────────────────────┐│
│  │ 代码合成 → Wasm / 动态执行 → GPIO / I2C / SPI → 持久化                 ││
│  └─────────────────────────────────────────────────────────────────────────┘│
│                                                                             │
│  gRPC/nanoRPC服务器 ◄──► 外设（GPIO、I2C、SPI、传感器、执行器）             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**工作流程：**
1. 用户发送WhatsApp：*"打开13号引脚的LED"*
2. ZeroClaw获取特定于开发板的文档（例如ESP32 GPIO映射）
3. LLM合成Rust代码
4. 代码在沙箱中运行（Wasm或动态链接）
5. GPIO被切换；结果返回给用户
6. 优化代码被持久化以供未来的"打开LED"请求使用

**全部在设备上发生。** 不需要主机。
